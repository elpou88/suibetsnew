Walrus Frontend Integration

// Frontend hook for Walrus operations
export function useWalrusProtocol() {
  // Store bet in Walrus after blockchain confirmation
  const storeBetInWalrus = async (bet) => {
    const walrusClient = new WalrusClient(
      process.env.WALRUS_RPC_URL
    );

    // Encode bet data
    const betData = JSON.stringify(bet);
    const encoder = new TextEncoder();
    const betBlob = encoder.encode(betData);

    // Upload to Walrus
    const storageProof = await walrusClient.store(
      betBlob,
      { duration_epochs: 100 } // 90+ day storage
    );

    // Get Walrus Object ID (immutable reference)
    const walrusObjectId = storageProof.object_id;

    // Store reference on Sui blockchain
    const tx = new Transaction();
    tx.moveCall({
      target: '0x...::betting::record_walrus_proof',
      arguments: [
        tx.object(betId),
        tx.pure.string(walrusObjectId),
      ],
    });

    return { walrusObjectId, txHash: result.digest };
  };

  // Retrieve bet from Walrus
  const retrieveBetFromWalrus = async (walrusObjectId) => {
    const walrusClient = new WalrusClient(
      process.env.WALRUS_RPC_URL
    );

    const betData = await walrusClient.read(walrusObjectId);
    return JSON.parse(new TextDecoder().decode(betData));
  };

  // Verify Walrus proof
  const verifyWalrusProof = async (walrusObjectId) => {
    const isValid = await walrusClient.verifyProof(
      walrusObjectId
    );
    return isValid; // Proves data hasn't been tampered
  };

  return {
    storeBetInWalrus,
    retrieveBetFromWalrus,
    verifyWalrusProof,
  };
}

// Smart contract linking Sui bets to Walrus storage
module suibets::walrus_integration {
  use sui::object::{Self, UID};
  use std::string::String;

  /// Bet record with Walrus reference
  public struct WalrusBetRecord has key, store {
    id: UID,
    bet_id: String,
    walrus_object_id: String,  // Reference to Walrus storage
    sui_tx_hash: String,       // Sui blockchain transaction
    storage_epoch: u64,        // When stored in Walrus
    verified: bool,
  }

  /// Store Walrus proof on Sui blockchain
  public entry fun record_walrus_proof(
    bet: &mut Bet,
    walrus_object_id: String,
    ctx: &mut TxContext
  ) {
    // Create immutable record linking bet to Walrus
    let record = WalrusBetRecord {
      id: object::new(ctx),
      bet_id: bet.id,
      walrus_object_id,
      sui_tx_hash: tx_context::digest(ctx),
      storage_epoch: tx_context::epoch(ctx),
      verified: true,
    };

    transfer::share_object(record);
  }

  /// Verify Walrus data matches Sui record
  public fun verify_walrus_integrity(
    bet: &Bet,
    record: &WalrusBetRecord
  ): bool {
    // Merkle proof verification against Walrus root
    // If hashes match -> data integrity confirmed
    bet.id == record.bet_id && record.verified
  }
}

6. Anti-Cheat System - HMAC-SHA256

Prevents oracle/admin from tampering with settlement results:

üõ°Ô∏è Anti-Cheat Flow
  1. Admin generates HMAC: HMAC-SHA256(betId|eventId|outcome|result, ADMIN_SECRET)
  2. Hash submitted with settlement transaction to smart contract
  3. Smart contract recalculates expected hash
  4. If hashes don't match ‚Üí SETTLEMENT FAILS
  5. Prevents result manipulation completely

Attack Prevention Example

9. Placing Bets - Complete Flow

Step-by-Step Process

  1. User selects sporting event from API-Sports data
  2. Frontend validates user has sufficient balance (from Sui blockchain)
  3. Transaction created with bet parameters
  4. Sui wallet signs the transaction (user approval required)
  5. Smart contract executes:
    • Validates bet parameters (amount > 0, odds valid)
    • Locks tokens in betting pool (atomic operation)
    • Creates immutable bet record on Sui
    • Sets status to "pending"
    • Returns bet ID
  6. Walrus stores bet data with storage proof
  7. User sees confirmation with Sui transaction hash
  8. Balance updated - funds now locked in contract

On-Chain Bet Lifecycle

Status Flow:
pending ‚Üí [event concludes]
         ‚Üí won (payout issued) / lost (loss recorded)
         ‚Üí claimed (user collected winnings)

OR

pending ‚Üí [admin refund]
         ‚Üí cancelled (refund issued)

10. Claiming Winnings

Automatic Settlement Process

  1. Event concludes in real-world (e.g., Liverpool 3-1 AFC Bournemouth)
  2. Oracle detects official final result via API-Sports
  3. Backend processes all pending bets for that event
  4. For each bet:
    • Calculate if bet won or lost
    • Generate HMAC-SHA256 hash for anti-cheat
    • Call smart contract settle_bet() function
    • Smart contract verifies hash validity
    • If won: transfer payout to user's wallet
    • If lost: mark as lost (tokens stay in pool as platform revenue)
  5. Bet status changed to "won" or "lost" on Sui blockchain
  6. Settlement transaction recorded in Walrus
  7. User sees "Won Bets" section in dApp UI
  8. User clicks "Claim Winnings" to confirm receipt
  9. User wallet receives payout instantly

14. Walrus Advanced Features

Merkle Tree Proofs

All bets are organized in a Merkle tree structure on Walrus, allowing users to prove specific bets exist without exposing entire bet history:

  • User proves: "I placed a bet on 2025-01-15"
  • Provides only: Merkle proof branch (~256 bytes)
  • Verifier confirms: Bet exists in root stored on Sui
  • Privacy: Other bets remain hidden

Zero-Knowledge Proofs

  • User proves: "I won $1000 in winnings"
  • Without revealing: Which bets, event IDs, or other details
  • Walrus verifies proof against immutable storage
  • Useful for privacy-preserving integrations
